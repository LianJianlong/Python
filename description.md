# Multable-B
## 多维表格 Web 应用（类似 NocoDB/Superset）、数据库储存、AI支持、用户管理、云部署

# 技术选型
- 前端框架：React
- 后端框架：FastAPI
- 数据库：PostgreSQL 或 MySQL
- 前后端分离架构
- API 规范：RESTful
- 用户认证：OAuth2
- 云部署支持：可 Docker 部署，也可部署在阿里云等平台

# 程序功能

## 主要功能
- 多维表格管理（类 NocoDB/Superset 表格、支持筛选、排序、聚合、归类、公式自定义、列粒度权限控制）
- 项目/部门/文件夹/表格/字段结构自定义（高权限账号可创建和管理，普通用户仅能在分配范围内操作）
- 表格视图：列表、日历、Gantt、照片墙等多视图切换，界面参考 superset/nocodb，支持自定义 CSS
- 表与表之间数据可相互引用，支持主键/外键设定
- 项目监控与综合看板，支持多类型数据可视化和自定义展示
- 项目工具模块：高级用户编写专业数据计算与可视化工具，普通用户仅能数据录入，不能更改模版、格式与计算方法
- 项目知识库：文件台账、项目规范、图纸、方案、过程文件（如来往信件、验收、安全、质量等分门别类文件，仅存链接，实际文件在云盘或服务器）
- AI assistant：数据库数据为本地知识库，结合联网搜索，具备 Chatbot 与数据分析型能力，自动推送消息与任务提醒、分配、错误等，接口开放
- 用户管理、角色管理、权限管理，权限粒度到列、超级管理员拥有全部权限，管理员仅能管理其下普通用户权限，普通用户仅能管理分配部分的内容，访客仅能查看分配表格，不可更改
- 增加内部邮箱系统，不同用户可互相发送邮件，AI assistant 可自动推送系统重要消息及任务等
- 历史记录：每个用户的修改与操作均保存，支持按操作次数或时间范围查询，记录存数据库
- 软件界面语言：默认中文，可选英文
- 个性化设置：字体、字号、背景、数据配置等
- 文件管理：仅存储文件云盘/服务器链接

## 模块大纲

### 1. 项目信息模块
- 一级文件夹（项目名称、自定义图标）
- 二级文件夹（部门名称、自定义图标）
- 三级表格（自定义名称，可设置主键、外键，表头筛选/排序/聚合/归类）
- 多视图支持（列表、日历、Gantt、照片墙）
- 表间数据互相引用

### 2. 项目监控模块
- 综合看板
- 多类型自定义图示、数据选择

### 3. 项目工具模块
- 工具仅由高级权限用户编写
- 普通用户只能信息录入，不可更改结构逻辑

### 4. 项目知识库模块
- 项目有效文件清单（台账）、规范、方案、图纸
- 来往信件、验收、安全、质量等过程文件
- 文件仅存储链接

### 5. AI assistant
- 以数据库作为本地知识库
- 支持联网数据检索
- 支持 Chatbot 和数据分析型助理
- 自动系统消息、任务推送
- 与内部邮件系统打通

### 6. 历史记录
- 每用户全部操作、修改记录
- 最近1000次或2个月操作可查
- 记录存于数据库

### 7. 用户、角色与权限管理
- 用户、角色管理，支持 200+ 用户
- 超级管理员（全部权限、权限粒度到列）
- 管理员（不可管理其他管理员，拥有自己下属普通用户的全部权限，可分配列权限）
- 普通用户（只能管理被分配文件夹/表格/视图的增删改查，无设计权限）
- 访客（仅查看分配的表格内容，无增删改权限）
- 列权限可分配到具体用户
- 用户邮箱及内部消息发送功能

### 8. 设置
- 语言设置（中文/英文）
- 字体、字号、背景色自定义
- 数据库与部署配置

# 其他说明
- UI 风格参考 NocoDB 和 Superset，支持自定义 CSS
- 可 Docker 部署或云部署
- 实际文件仅存储链接，文件归档至云盘或服务器
- 高权限用户可创建项目/部门/表格/字段，普通用户不可变更结构
